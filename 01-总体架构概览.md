## 一、项目概述

Hermes Agent 是一个**自进化的 AI 代理（Self-improving AI Agent）**，由 Nous Research 构建。其核心定位是：内置学习闭环的智能代理——从经验中创建技能，在使用中改进技能，主动持久化知识，搜索过往对话，并在跨会话中逐步构建对用户的深度理解。

### 1.1 核心技术能力

| 能力 | 描述 |
|------|------|
| **自进化学习** | 每次对话后自动审查（Background Review），从中提取技能和记忆 |
| **多模型支持** | 通过传输抽象层支持 16+ OpenAI 兼容提供商、Anthropic、Bedrock、Codex |
| **多平台消息** | Telegram、Discord、Slack、WhatsApp、Signal、WeCom、Feishu 等 |
| **终端运行** | 六种后端：本地、Docker、SSH、Daytona、Singularity 和 Modal |
| **工具系统** | 80+ 内置工具 + MCP 协议外部工具支持 |
| **技能市场** | 10 个源适配器的技能 Hub，支持 agentskills.io 开放标准 |
| **定时自动化** | 内置 cron 调度器，跨平台投递 |
| **子代理委派** | 生成隔离子代理处理并行工作流 |

### 3.3 传输抽象层

传输层位于 `agent/transports/`，是 **桥接模式**的核心实现：

```
ProviderTransport (ABC)
├── ChatCompletionsTransport   — api_mode="chat_completions"（16+ 提供商）
├── AnthropicTransport         — api_mode="anthropic_messages"
├── ResponsesApiTransport      — api_mode="codex_responses"（GPT-5.x）
└── BedrockTransport           — api_mode="bedrock_converse"
```

统一接口：
- `convert_messages()` → OpenAI 消息 → 提供商原生格式
- `convert_tools()` → OpenAI 工具定义 → 提供商原生格式
- `build_kwargs()` → 构建完整 API 调用参数字典
- `normalize_response()` → 原生响应 → `NormalizedResponse` 统一类型
- `extract_cache_stats()` → 提取缓存统计
- `map_finish_reason()` → 停止原因映射

## 五、关键设计原则

### 5.1 代码组织原则

1. **关注点分离**：传输层（数据形状）与 Agent（生命周期）严格分离
2. **单一真相源**：工具注册表是所有工具元数据的唯一权威
3. **薄编排层**：`model_tools.py` 只做发现/过滤/分发，不持有数据
4. **延迟导入**：所有重依赖（anthropic、boto3、azure-identity）按需加载
5. **向后兼容**：通过 `_ra()` 模式保持 `run_agent` 的 patch 契约

### 5.2 安全设计原则

1. **纵深防御**：结构检查 → 模式匹配 → AST 审计 → 隔离区安装
2. **最小权限**：子代理工具白名单（delegate_task 阻止递归委托/内存写入/消息发送）
3. **不可绕过**：Hardline 模式（rm -rf /、mkfs、fork bomb）无法通过 --yolo 绕过
4. **安全回滚**：技能创建使用原子写入，审核失败可完整回滚
5. **凭证剥离**：MCP 工具错误消息自动剥离 API keys

### 5.3 性能优化原则

1. **前缀缓存复用**：系统提示词"字节稳定"设计，三层结构（稳定/上下文/易变）
2. **双层技能缓存**：LRU 内存缓存 + 磁盘快照（避免每次 rebuild 的文件 I/O）
3. **WAL 模式 SQLite**：并发读 + 单写者
4. **持久事件循环**：避免每个工具调用都创建/销毁 asyncio 循环
5. **并行工具执行**：ThreadPoolExecutor 并发（最多 8 worker）

```python

class AIAgent:
    def __init__(self, model, base_url, provider, ...):  # 60+ 参数
        # 1. Provider 自动检测（从 model/base_url/provider 推断）
        # 2. API mode 确定（chat_completions/anthopic_messages/codex_responses/bedrock_converse）
```

```python
def interruptible_api_call(agent, api_kwargs):
    if agent.api_mode == "codex_responses":
        return _run_codex_stream(agent, api_kwargs)
    elif agent.api_mode == "anthropic_messages":
        return _anthropic_messages_create(agent, api_kwargs)
    elif agent.api_mode == "bedrock_converse":
        return _bedrock_converse(agent, api_kwargs)
```

## 八、总结

Hermes Agent 是一个架构设计精良的大型 Python 项目，其核心特征包括：

1. **分层清晰**：入口层 → CLI层 → Agent核心层 → 工具系统层 → 网关层 → 数据持久化层
2. **抽象得当**：传输层（ProviderTransport ABC）、上下文引擎（ContextEngine ABC）、内存 provider（MemoryProvider ABC）、平台适配器（BasePlatformAdapter ABC）
3. **安全优先**：多层防御（Hardline → 危险模式 → 智能审批 → 交互确认），技能安全扫描，凭证剥离
4. **性能意识**：三层系统提示词缓存、双层技能索引缓存、WAL SQLite、持久事件循环
5. **自进化闭环**：Background Review → Skill 创建/改进 → Curator 管理 → 系统提示词索引 → 未来对话受益
6. **可扩展性**：MCP 协议、插件系统、技能 Hub、平台 registry、Provider 传输注册表
