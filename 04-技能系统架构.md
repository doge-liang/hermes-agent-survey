# Hermes Agent 技能系统架构深度分析

## 一、技能生命周期

```
创建 (agent自主/Hub安装/捆绑同步)
  → 发现 (双层缓存：LRU内存 + 磁盘快照)
  → 索引 (构建系统提示词中的技能表)
  → 加载 (/skill-name 命令 或 LLM自主调用)
  → 使用 (bump_use 记录遥测)
  → 改进 (Background Review → skill_manage patch)
  → 归档 (Curator: active → stale → archived)
```

### 1.1 创建来源

**A. Agent自主创建** — `tools/skill_manager_tool.py`：
- `create`：创建目录，写入SKILL.md（YAML头 + markdown正文）
- `edit`：完整重写SKILL.md
- `patch`：模糊匹配引擎定向查找替换
- `delete`：移除目录，支持 `absorbed_into` 声明合并
- `write_file`/`remove_file`：管理辅助文件

**B. Hub安装** — `tools/skills_hub.py`（10个源适配器）：
- 上传流程：search → fetch bundle → quarantine → 安全扫描 → install
- 源适配器：HermesIndex、skills.sh、GitHub、clawhub.ai、LobeHub等
- 信任仓库：openai、anthropics、huggingface、NVIDIA

**C. 捆绑同步** — `tools/skills_sync.py`：通过 `.bundled_manifest` 将仓库自带技能写入 `~/.hermes/skills/`

### 1.2 发现与索引

**双层缓存策略：**
1. 进程内LRU缓存：键由(skills_dir, external_dirs, available_tools, ...)组成，容量8条
2. 磁盘快照：`~/.hermes/.skills_prompt_snapshot.json`，按mtime/size文件清单校验有效性

### 1.3 加载

`agent/skill_commands.py` 处理斜杠命令调用：
1. 解析 `/skill-name --instruction` → cmd_key
2. 通过 `_load_skill_payload` 加载技能内容
3. 构建结构化消息：激活说明 + 完整技能正文 + skill目录路径 + 配置值 + 辅助文件列表
4. 调用 `skill_usage.bump_use()` 进行遥测

`agent/skill_preprocessing.py` 提供模板变量替换和inline-shell展开。

## 二、磁盘物理结构

```
~/.hermes/skills/
├── my-skill/
│   ├── SKILL.md          # YAML头 (name, description, platforms...) + markdown
│   ├── references/       # 辅助文档
│   ├── templates/        # 模板文件
│   ├── scripts/          # 可执行脚本
│   └── assets/           # 静态资源
├── .usage.json           # Curator遥测
├── .bundled_manifest     # 内置技能同步清单
├── .hub/
│   ├── lock.json         # Hub安装来源记录
│   ├── quarantine/       # 安装前安全隔离区
│   ├── audit.log         # 审计日志
│   └── taps.json         # 自定义GitHub taps
├── .curator_suppressed   # 已删除内置技能抑制列表
└── .archive/             # 归档技能（可恢复）
```

## 三、安全体系

### 3.1 Skills Guard (~130种正则模式)

**多层防御：**
1. 结构检查：文件数>50、总大小>1MiB、二进制扩展名、符号链接、可执行权限位
2. 正则威胁模式（6类）：数据窃取、提示注入、破坏性操作、持久化、混淆、供应链
3. 隐形Unicode检测：16种隐形字符（零宽空格、双向覆盖、格式隔离字符等）
4. `.skillignore` 支持：gitignore风格排除

**安装策略矩阵：**

| 来源 \ 判定 | safe | caution | dangerous |
|---|---|---|---|
| builtin | allow | allow | allow |
| trusted | allow | allow | block |
| community | allow | block | block |

### 3.2 AST深度审计

扫描Python文件中的 `importlib.import_module()`、动态 `__import__`、计算属性 `getattr`、`__dict__[<computed>]` 等反射模式。

### 3.3 Agent创建技能的安全扫描

- 前台创建：安全扫描默认关闭
- `skills.guard_agent_created = true` 时，`dangerous`/`caution` 判定触发回滚
- 写入前通过临时原子写入保证安全回滚

## 四、Curator策展器

### 4.1 核心职责

- 仅触及agent创建的技能（`is_agent_created()`）
- 绝不自动删除——仅归档
- 固定（pinned）技能绕过所有自动状态转换
- 使用辅助客户端；不触碰主会话的prompt缓存

### 4.2 自动状态转换（纯函数，无LLM）

```
active ──[last_activity <= stale_after_days]──→ stale
stale  ──[last_activity > stale_after_days]──→ active (已重新激活)
stale  ──[last_activity <= archive_after_days]──→ archived
```

默认值：`stale_after_days=30`，`archive_after_days=90`

### 4.3 LLM审查Pass — 伞形构建合并

核心理念："人类维护者会将其编写为N个独立技能还是1个含有N个小节的技能？"

三种合并方式：
1. 合并到现有伞形 → patch + 归档同级
2. 创建新伞形SKILL.md
3. 降级为引用/模板/脚本 → 移至子目录 + 归档原始技能

### 4.4 分类调和

合并三种信号来源：
1. **模型声明的 `absorbed_into`**（权威——模型删除时告知）
2. **模型的结构化YAML块**（意图和理由的权威）
3. **工具调用启发式**（真实审计，捕捉幻觉和遗漏）

### 4.5 每运行报告 + 快照 + 回滚

- 写入 `logs/curator/{YYYYMMDD-HHMMSS}/run.json` + `REPORT.md`
- 变更前创建 `~/.hermes/skills/` 的tar.gz快照
- `rollback()`: 当前状态安全快照 → 提取选定快照 → 恢复cron作业技能链接

## 五、Skill Bundles (技能捆绑包)

复合技能别名：`~/.hermes/skill-bundles/*.yaml`

```yaml
name: backend-dev
skills:
  - github-code-review
  - test-driven-development
  - github-pr-workflow
instruction: |
  Optional extra guidance
```

- Bundle和skill共享同一斜杠名称时，bundle优先
- 若bundle中某个参照技能不存在，跳过并提示，bundle仍加载
- 每个技能都通过 `bump_use` 记录遥测

## 六、背景审查触发技能改进

1. **来源标记**：`skill_provenance.py` 通过 `ContextVar` 区分前台 vs 后台审查。
2. **遥测收集**：每次查看/使用/修补后更新 `.usage.json`，通过文件锁序列化。
3. **系统提示指引**：`SKILLS_GUIDANCE` 注入系统提示头部。
4. **缓存失效**：`skill_manage` 成功后调用 `clear_skills_system_prompt_cache(clear_snapshot=True)`。
5. **创建提醒**：`skills.creation_nudge_interval`（默认10轮）控制提示频率。
